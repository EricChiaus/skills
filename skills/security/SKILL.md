# Security Skill

This skill provides guidance and best practices for implementing security in software projects. It covers topics such as authentication, authorization, secure coding, data protection, vulnerability management, and compliance.

## When to use this skill
- When you need to add security features to your application
- When reviewing code for security vulnerabilities
- When designing authentication and authorization systems
- When handling sensitive data or implementing encryption
- When ensuring compliance with security standards

## Key Topics
- Authentication (passwords, OAuth, SSO, MFA)
- Authorization (RBAC, ABAC, least privilege)
- Secure coding practices (input validation, output encoding, avoiding injection)
- Data protection (encryption at rest and in transit, key management)
- Vulnerability management (dependency scanning, patching)
- Security testing (static/dynamic analysis, penetration testing)
- Compliance (GDPR, HIPAA, PCI DSS)

## Usage
Use this skill when you need to:
- Secure user authentication and session management
- Protect sensitive data
- Prevent common vulnerabilities (XSS, CSRF, SQL injection, etc.)
- Implement secure communication (TLS/SSL)
- Review or audit code for security issues
